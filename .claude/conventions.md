# Conventions

Naming
- front-end: `camelCase` for variables and hooks, `PascalCase` for components
- backend: `camelCase` methods, `PascalCase` classes
- files: `kebab-case` for routes and util modules

Styling
- no dark theme by default
- mobile-first breakpoints
- use MUI theming for colors/layout

Performance
- marketplace list: virtualization + pagination
- model list: cached with TTL

Token rules
- no sending entire repo or huge logs in chat
- summarizing only changed files
